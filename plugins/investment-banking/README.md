# Investment Banking

Turn research and diligence into client-ready pitch materials, recommendations, models, and transaction analysis.

## When to use this plugin

Use Investment Banking when you need to turn research and diligence into client-ready materials or analysis for transaction execution and coverage work: sell-side processes, CIMs, buyer lists, valuations, merger and LBO models, capital-markets options, financing analysis, pitch materials, and deal tracking. You can start with uploaded files, pasted context, connected sources, an existing workbook, or a clear mandate question.

## Get started

Ask Codex:

`@Investment Banking Help me get started`

This optional setup flow helps you save useful output and audience defaults, choose the tools and sources to use, decide whether to add a weekly source check, and pick a first workflow.

You can skip setup and start any workflow directly.

## Example workflows

| Workflow | Try this | Skill | Result |
| --- | --- | --- | --- |
| Create a company tearsheet | `Build a banker-facing tearsheet for this target company` | `company-tearsheet` | A source-backed profile for coverage, screening, meeting context, or deal work |
| Prepare for a meeting | `Prepare a brief and question list for tomorrow's management meeting` | `meeting-prep` | A focused meeting brief, question set, and follow-up action plan |
| Build a buyer or investor list | `Build a prioritized buyer universe for this sell-side process` | `buyer-investor-list` | A rationale-backed universe with outreach waves and tracker-ready parties |
| Draft a CIM | `Draft a CIM storyboard and first-pass narrative for this business` | `cim-builder` | A buyer-facing CIM, teaser, lender presentation, or management-presentation draft |
| Review a CIM or banker deck | `Tear down this CIM and identify the biggest diligence gaps and model handoffs` | `cim-teardown` | A structured view of claims, gaps, red flags, and next analytical steps |
| Build a comps valuation | `Build a trading comps valuation for this company and pressure-test the peer set` | `comps-valuation` | A source-backed comps report or workbook with multiples, implied valuation, and EV bridge |
| Build a DCF | `Build a DCF with sensitivities and an EV-to-equity bridge` | `dcf-model-builder` | A DCF workbook with WACC, terminal value, sensitivities, and a valuation bridge |
| Build a three-statement model | `Build an integrated three-statement operating model for this company` | `three-statement-model-builder` | A linked operating-model export with statements, drivers, checks, and scenarios |
| Build a merger model | `Build an accretion and dilution model for this proposed transaction` | `merger-model-builder` | A strategic M&A model covering consideration, ownership, synergies, purchase accounting, financing, and EPS impact |
| Build an LBO model | `Build an LBO model for this sponsor take-private scenario` | `lbo-model-build` | A sponsor model covering sources and uses, debt, sweep, liquidity, returns, and downside |
| Frame a capital-markets issuance | `Compare financing options for this issuer and recommend the best path` | `capital-markets-issuance` | An execution view across ECM, DCM, private placements, market window, investor targeting, and use of proceeds |
| Underwrite a private-credit case | `Build a lender-side underwriting view for this borrower` | `private-credit-underwriting` | A credit decision package covering debt sizing, downside, liquidity, collateral, recovery, and proceed-or-decline posture |
| Analyze a covenant package | `Review these credit documents and summarize covenant headroom and leakage risks` | `covenant-package-analyzer` | A finance-side covenant review covering definitions, baskets, leakage, headroom, amendments, and waivers |
| Analyze a distressed recovery | `Build a recovery waterfall for this distressed capital structure` | `distressed-recovery-waterfall` | A recovery analysis across claims, lien priority, fulcrum security, plan value, liquidation value, and sale paths |
| Normalize deal financials | `Turn these messy deal financials into a model-ready source pack` | `financials-normalizer` | Normalized statements, KPI schedules, source maps, and QA flags |
| Build a sensitivity framework | `Pressure-test the key drivers and downside cases in this analysis` | `scenario-sensitivity-generator` | A scenario, sensitivity, stress-test, or breakeven framework for decision thresholds |
| Draft a pitch deck | `Build a page plan and first-pass content for this client pitch` | `pitch-deck-builder` | A banker-ready pitch outline, page plan, and draft slide content |
| Draft a memo | `Turn this analysis into a concise committee memo` | `memo-builder` | A client, committee, board, financing, process, or diligence note grounded in existing analysis |
| Track a deal process | `Build a tracker for buyer outreach, NDAs, diligence, bids, and deadlines` | `deal-process-tracker` | A banker-facing workbook for process status, chronology, and next actions |
| Audit a model | `Audit this model for formula, source, assumption, and sensitivity issues` | `model-audit-tieout` | A readiness review with the highest-impact issues and remediation path |
| Run deck or report QC | `QC this pitch deck before circulation` | `ib-deck-qc` | A circulation-focused check of numbers, units, sources, charts, footnotes, formatting, and page takeaways |

## Integrations

Investment Banking can use available tools when they are connected:

| Source | Supported integrations | What they unlock |
| --- | --- | --- |
| Market and company data | FactSet, LSEG, S&P, Daloopa, Quartr, Moody's | Source-backed company profiles, financial analysis, market context, transcripts, and credit evidence |
| Private-market and expert context | PitchBook, Third Bridge | Buyer, sponsor, investor, lender, market, and diligence context when available |
| Collaboration and documents | Slack, Google Drive, Gmail | Mandate context, source documents, review inputs, and controlled handoff |

Connected sources improve grounding, but you can also begin with uploaded files, exported data, pasted excerpts, existing workbooks, and manually provided assumptions.
